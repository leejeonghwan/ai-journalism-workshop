2-1. 프롬프트의 기본 구조 (10분)

나쁜 프롬프트 vs 좋은 프롬프트

나쁜 예 좋은 예
"경제 기사 써줘" 아래 보도자료를 역피라미드 구조의 스트레이트 기사 800자로 작성하라. 홍보성 표현("획기적", "최초")을 제거하고, 주어를 명확히 하라. 빠진 정보는 [확인 필요]로 표기하라.
"이 데이터 분석해줘" 첨부 CSV의 2020-2024년 추이를 분석하라. (1) 변화율이 가장 큰 항목, (2) 지역별 편차, (3) 이상치를 찾고, 각 수치를 포함하라. 인과관계를 단정짓지 말고 상관관계만 서술하라.

프롬프트 4요소 공식:

역할(Role) + 맥락(Context) + 지시(Instruction) + 형식(Format)

예시:

당신은 10년차 경제부 기자입니다. (역할)
한국은행이 오늘 기준금리를 동결했습니다. (맥락)
이 결정의 배경과 영향을 해설 기사로 작성하세요. (지시)
800자, 소제목 3개, 전문가 인용이 필요한 곳은 [인용 필요]로 표기하세요. (형식)

2-2. 실습 A: 기사 요약 프롬프트 비교 (15분)

[실습 자료 배포] 실제 기사 1건 (1500자 내외)

미션: 같은 기사를 5가지 프롬프트로 요약하고 결과를 비교하라.

프롬프트 1 (최소): "이 기사를 요약해줘"
프롬프트 2 (분량 지정): "이 기사를 3문장으로 요약해줘"
프롬프트 3 (구조 지정): "핵심 사실 / 배경 / 영향으로 나눠서 요약해줘"
프롬프트 4 (관점 지정): "당신은 편집국장이다. 이 기사의 뉴스 가치를 평가하고 부족한 점을 지적하라"
프롬프트 5 (전문): 아래 기사를 읽고 다음 형식으로 요약하라.
  - 핵심 사실 (1~2문장): 누가, 무엇을, 왜
  - 배경 맥락 (2~3문장): 왜 지금 이것이 중요한가
  - 이해관계자: 찬성 입장 / 반대 입장 (각 1문장)
  - 후속 전개: 앞으로 주목할 포인트
  조건: 원문에 없는 정보를 추가하지 마라. 평가를 넣지 마라.

비교 포인트: 정확성, 빠진 정보, 과잉 해석, 실용성

2-3. 실습 B: 인터뷰 질문 생성 (10분)

미션: 다음 주 인터뷰 대상을 정하고 AI로 질문 10개를 생성하라.

나는 [주제]에 대해 [직함/이름]을 인터뷰할 예정이다.

배경 정보:
- [인터뷰이의 최근 발언/행동]
- [관련 이슈의 현재 상황]

다음 기준으로 인터뷰 질문 10개를 만들어라:
1. 도입 질문 2개 (라포 형성)
2. 핵심 질문 5개 (구체적 사실/입장 확인)
3. 도전적 질문 2개 (반론/비판 제기)
4. 마무리 질문 1개 (전망/메시지)

각 질문에 [왜 이 질문을 하는지] 의도를 병기하라.

토론: AI 질문 중 실제로 쓸 만한 것은? 기자가 추가해야 할 질문은?

2-4. 7가지 프롬프트 기법 (15분)

Paul Bradshaw(버밍엄시티대) 교수가 정리한 저널리스트를 위한 프롬프트 기법:

# 기법 설명 예시
1 역할 부여 AI에게 전문가 역할을 지정 "당신은 헌법학 교수입니다"
2 문서 기반 질의 (RAG) 자료를 직접 첨부하고 그 안에서만 답하게 함 NotebookLM에 보고서 업로드
3 단계적 사고 (Chain-of-Thought) 추론 과정을 보여달라고 요청 "단계별로 설명하며 결론을 도출하라"
4 형식 지정 출력 구조를 미리 정의 "표 형식으로 / JSON으로 / 역피라미드로"
5 제약 조건 금지 사항을 명시 "원문에 없는 정보를 추가하지 마라"
6 반복 개선 첫 결과를 피드백하여 재요청 "2번 항목을 더 구체적으로 다시 써줘"
7 메타 프롬프트 AI에게 프롬프트를 만들어달라고 요청 "이 작업에 최적의 프롬프트를 설계해줘"

3교시: 취재·보도에 AI 쓰기 (50분)

실습 중심 — 리서치, 기사 작성, 팩트체크

3-1. 실습 C: AI로 취재 리서치 (15분)

도구: Perplexity + ChatGPT/Claude

미션: 현재 자신이 취재 중인 주제(또는 관심 주제)로 배경 조사를 수행하라.

단계:

  1. (5분) Perplexity로 배경 조사

    • "2026년 [주제]의 핵심 쟁점 5가지와 각각의 이해관계자를 정리하라"
    • 결과에 딸린 출처 링크를 클릭하여 실제 존재하는 기사인지 확인

  2. (5분) Claude/ChatGPT로 보고서 분석

    • 관련 보고서(PDF)를 업로드하고 "핵심 발견 5가지를 원문 페이지 번호와 함께 정리하라"
    • 원문과 대조

  3. (5분) 교차 검증

    • Perplexity 결과와 직접 보고서 분석 결과를 비교
    • 불일치하는 부분 메모

핵심 교훈: AI는 리서치 "보조"이지 리서치 자체가 아니다. 출처를 반드시 직접 확인한다.

  1. (7분) AI로 초안 생성
다음 보도자료를 800자 내외의 스트레이트 기사로 작성하라.

[보도자료 전문]

조건:
- 역피라미드 구조
- 홍보성 표현 제거 ("획기적인", "최초의", "성공적인")
- 주어를 명확히 하라
- 빠진 정보: [확인 필요]로 표기
- 반대 의견이 필요한 부분: [보완 필요]로 표기
  1. (8분) AI 초안 편집 실습

참가자가 AI 초안을 직접 편집한다:

  • 홍보성 표현이 남아있지 않은가?
  • 주체가 명확한가? ("발표했다" → 누가?)
  • [확인 필요]를 AI가 적절히 표시했는가?
  • 인용문이 보도자료 원문과 일치하는가?
  • "결국", "사실상", "핵심은" 등 경계 표현이 있는가?
  1. (5분) 톤 변환 체험
  • 같은 내용을 뉴스레터 도입부(200자)로 변환
  • 같은 내용을 X(트위터) 스레드 5개로 변환

3-3. 실습 E: AI 팩트체크 (15분)

미션: 아래 3개 주장 중 1개를 선택하여 AI로 검증하라.

(강사가 미리 준비한 검증 대상 3건 — 사실/반쪽 사실/거짓 각 1건)

단계:

  1. (5분) AI 1차 검증
다음 주장이 사실인지 검증하라.

주장: "[구체적 주장]"

검증 방법:
1. 이 주장의 핵심 팩트를 분리하라
2. 각 팩트의 근거와 출처를 제시하라
3. 반박 근거가 있다면 함께 제시하라
4. 최종 판정: 사실 / 대체로 사실 / 절반의 사실 / 대체로 거짓 / 거짓

  1. (5분) Perplexity로 교차 검증

    • 같은 주장을 Perplexity에 입력
    • 제공된 출처 링크를 직접 확인

  2. (5분) 결과 비교 토론

    • AI가 "사실"이라고 한 것이 실제로 사실인가?
    • AI가 제시한 출처가 실존하는가?
    • 핵심: AI는 팩트체크 "도우미"이지 팩트체커가 아니다

4교시: 데이터 분석 + 콘텐츠 변환 (50분)

4-1. 실습 F: AI로 데이터 분석 (30분)

도구: ChatGPT (Code Interpreter) 또는 Claude (Artifacts) [실습 자료 배포] 공공데이터 CSV 1건 (예: 지역별 인구 변화, 범죄 통계, 대기오염 데이터)

단계:

  1. (5분) 데이터 탐색
첨부된 CSV 데이터를 분석하라.

1. 데이터 개요: 총 행/열 수, 기간, 주요 변수
2. 기술 통계: 주요 변수의 평균, 중위값, 최대/최소
3. 추이 분석: 시간에 따른 주요 변화 패턴
4. 이상치 탐지: 비정상적으로 높거나 낮은 값
5. 기사 앵글 제안: 이 데이터로 쓸 수 있는 기사 앵글 3개

각 분석에 구체적 수치를 포함하라. 인과관계를 단정짓지 마라.

  1. (10분) 심층 분석 질문

    • "변화율이 가장 큰 상위 5개 항목은?"
    • "지역별 편차가 가장 큰 지표는?"
    • "2020년 대비 2024년 역전 현상이 있는가?"

  2. (10분) 시각화

    • "이 추이를 꺾은선 그래프로 그려라"
    • "상위 10개를 수평 막대 차트로 보여달라"
    • 시각화의 정확성을 원본 데이터와 대조

  3. (5분) 기사 리드 작성

위 분석 결과 중 [가장 놀라운 발견]을 기사 리드 3가지 버전으로 써라.
버전 1: 스트레이트
버전 2: 피처 스타일
버전 3: 데이터 저널리즘 스타일

주의사항 강조:

  • AI 계산 결과를 엑셀로 반드시 샘플 검증 (3~5개 행)
  • 개인정보 포함 데이터는 업로드 금지
  • AI가 "인과관계"를 주장하면 의심하라 — 상관관계와 인과관계는 다르다

4-2. 실습 G: 기사 → 멀티플랫폼 콘텐츠 변환 (20분)

미션: 자신이 쓴 기사(또는 3교시에 편집한 기사)를 4가지 플랫폼용으로 변환하라.

1. 뉴스레터 도입부 (5분)

이 기사를 뉴스레터 도입부 200자로 바꿔라.
독자에게 말을 거는 톤으로, "왜 이것이 당신에게 중요한지"로 시작하라.
3가지 버전을 만들어라.

2. X(트위터) 스레드 (5분)

이 기사를 5개 트윗으로 구성된 X 스레드로 변환하라.
1번: 훅 (가장 놀라운 사실)
2-4번: 핵심 내용 (각 280자 이내)
5번: 시사점 + 원문 링크 안내

3. 인스타그램 캐러셀 텍스트 (5분)

이 기사를 인스타그램 캐러셀 카드 5장의 텍스트로 변환하라.
1장: 제목 (한 줄, 임팩트 있게)
2-4장: 핵심 내용 (각 50자 이내, 숫자/비교 강조)
5장: 핵심 메시지 + CTA

4. 60초 숏폼 스크립트 (5분)

이 기사를 60초 유튜브 쇼츠 스크립트로 변환하라.
[훅 - 5초]: 시청자를 잡는 첫 문장
[본문 - 45초]: 핵심 3가지
[마무리 - 10초]: 한줄 요약 + 구독 유도

편집 체크: AI가 팩트를 왜곡하거나 과장하지 않았는지 반드시 확인

5-3. AI 시대 기자의 핵심 역량 (10분)

AI가 대체할 수 없는 것

AI가 잘하는 것 AI가 못하는 것 (기자 고유 역량)
대량 자료 요약/분석 현장에 가서 보고 느끼기
패턴 탐지 취재원과 신뢰 관계 구축
초안 생성 "이것은 뉴스인가?" 판단
다국어 번역 윤리적 딜레마 판단
데이터 시각화 권력 감시 의지
형식 변환 맥락과 뉘앙스 이해

기자가 키워야 할 AI 역량

  1. 프롬프트 리터러시 — AI에게 정확히 요구하는 능력
  2. 검증 습관 — AI 산출물을 맹신하지 않는 비판적 태도
  3. 도구 선택 능력 — 상황에 맞는 AI 도구를 고르는 판단력
  4. 데이터 리터러시 — AI와 함께 데이터를 다루는 기본기
  5. 윤리적 판단 — AI 사용의 경계선을 아는 감각

  • "AI 시대에 저널리즘의 가치는 올라가는가, 내려가는가?"

6교시: AI 에이전트 뉴스룸 (50분)

"코딩으로 취재 파이프라인 만들기"

6-1. 에이전트 AI란 무엇인가 (10분)

지금까지 배운 AI 활용은 1회성 대화였다. 질문하고, 답 받고, 끝. 에이전트 AI는 다르다.

대화형 AI vs 에이전트 AI

대화형 AI (1~5교시) 에이전트 AI (6~7교시)
사람이 매번 지시 AI가 스스로 단계를 나눠 실행
1회 질문 → 1회 답변 목표를 주면 여러 단계를 자율 수행
도구 1개 사용 웹 검색, 파일 읽기, 코드 실행을 조합
"이 기사 요약해줘" "이 주제를 조사하고, 데이터를 분석하고, 기사 초안을 써줘"

뉴스룸에 비유하면:

  • 대화형 AI = 인턴에게 한 가지씩 시키는 것
  • 에이전트 AI = 취재팀을 꾸려서 미션을 주는 것

2026년 Reuters Institute 전문가 17인이 꼽은 핵심 트렌드 1위가 에이전트 AI다. "뉴스룸이 에이전트 AI로 복잡한 워크플로우를 자동화하기 시작할 것."

6-2. Claude Code 시연 — 터미널에서 AI와 일하기 (15분)

[시연] Claude Code란?

Claude Code는 터미널(명령줄)에서 작동하는 AI 코딩 도구다. 일반 ChatGPT와 다른 점:

ChatGPT/Claude 웹 Claude Code
브라우저에서 대화 터미널에서 실행
파일을 수동 업로드 내 컴퓨터의 파일을 직접 읽고 쓴다
결과를 복사-붙여넣기 결과를 파일로 저장, 스크립트 실행
한 번에 하나의 작업 여러 에이전트를 병렬로 실행

[라이브 시연]

# Claude Code 실행
claude

# 대화 모드에서 바로 작업 지시
> 오늘 발행된 정부 보도자료 3건을 요약하고, 
  공통 키워드를 뽑아서 표로 정리해줘.
  결과를 briefing.md에 저장해.

Claude Code가 스스로:

  1. 웹을 검색하여 보도자료를 찾고
  2. 각 보도자료를 요약하고
  3. 공통 키워드를 추출하여 표를 만들고
  4. briefing.md 파일로 저장한다

핵심: 기자가 "무엇을 원하는지"만 말하면, AI가 "어떻게 할지"를 스스로 결정한다.

6-3. 하네스 뉴스룸 — 에이전트 팀 아키텍처 (10분)

[시연] 에이전트 팀이란?

하네스(Harness)는 전문 에이전트들이 역할을 나눠 협업하는 시스템이다.

desk(편집국장) → beat-reporter(취재) → analyst(분석) → writer(집필) → factchecker(검증) → editor(편집)

실제 뉴스룸과 똑같은 구조:

에이전트 뉴스룸 역할 하는 일
desk 편집국장 이슈 선정, 취재 지시, 앵글 결정
beat-reporter 취재기자 자료 수집, 출처 구조화, 취재 노트
analyst 분석기자 패턴 발견, 맥락 연결, 인사이트 도출
writer 기사작성자 초안 집필, 구조화, 톤 조절
factchecker 팩트체커 인용 검증, 숫자 확인, 출처 대조
editor 편집자 문체 교정, 헤드라인, 최종 점검

각 에이전트는 파일로 정의된다:

# .claude/agents/desk.md

---
name: desk
description: "편집국장. 이슈 선정과 취재 브리프를 작성한다."
---

당신은 편집국장입니다.

## 핵심 역할
1. 원자료에서 뉴스 가치가 있는 이슈를 선별
2. 취재 방향과 앵글을 결정
3. 취재기자에게 브리프를 전달

## 작업 원칙
- 팩트와 주장을 분리한다
- 출처 없는 평가 문장은 삭제한다
- "결국", "본질은" 같은 경계 표현을 쓰지 않는다

[시연] 에이전트가 산출물을 넘겨주는 방식

_workspace/
├── desk_brief.md          ← desk가 작성 → beat-reporter가 읽음
├── reporter_notes.md      ← beat-reporter 작성 → analyst가 읽음
├── insight_report.md      ← analyst 작성 → writer가 읽음
├── draft.md               ← writer 작성 → factchecker가 읽음
├── factcheck_result.md    ← factchecker 작성 → editor가 읽음
└── final_article.md       ← editor 작성 → 최종 산출물

각 에이전트는 이전 에이전트의 파일을 읽고, 자기 결과를 파일로 남긴다. 뉴스룸의 원고 전달 시스템과 동일한 구조다.

6-4. 실습 I: 에이전트 정의서 작성 (15분)

미션: 자신의 취재 분야에 맞는 에이전트 1개를 직접 설계하라.

7교시: 실전 에이전트 실습 (50분)

"자료 조사부터 기사 초안까지"

7-1. 실습 J: 에이전트로 자료 조사 (15분)

[라이브 시연 + 따라하기]

실제 뉴스 이슈를 하나 정하고, 에이전트 4개를 병렬로 보내 조사한다.

시나리오: "AI 규제 법안 국제 비교" 기사를 쓴다고 가정

나: 다음 4개 영역을 병렬로 조사해줘.

에이전트 1 — EU AI Act
  EU AI Act의 핵심 조항을 정리하라.
  고위험 AI 분류 기준, 벌금 규정, 시행 일정을 포함.
  출처 URL을 반드시 붙여라.
  결과를 _workspace/research_eu.md에 저장.

에이전트 2 — 미국 AI 규제
  미국의 AI 규제 현황을 정리하라.
  연방 차원 행정명령, 주별 법안(캘리포니아, 콜로라도 등),
  의회 법안 발의 현황을 포함.
  결과를 _workspace/research_us.md에 저장.

에이전트 3 — 한국 AI 기본법
  한국 AI 기본법의 핵심 내용을 정리하라.
  시행 시기, 주요 조항, 해외 법안과의 차이점을 포함.
  결과를 _workspace/research_kr.md에 저장.

에이전트 4 — 일본·중국 AI 규제
  일본과 중국의 AI 규제 접근 방식을 비교하라.
  일본의 자율 규제 모델, 중국의 생성형 AI 관리법을 포함.
  결과를 _workspace/research_asia.md에 저장.

[시연 포인트]

  • 4개 에이전트가 동시에 웹을 검색하고 자료를 수집하는 모습
  • 각 에이전트가 자기 영역의 파일을 _workspace/에 저장
  • 사람이 기다리는 동안 4명이 동시에 일하는 효율

참가자 실습:

  • 자신의 취재 분야에서 이슈 1개를 선택
  • 3~4개 조사 영역으로 나눠서 에이전트에게 지시
  • 결과 파일들이 생성되는 것을 확인

7-2. 실습 K: 에이전트로 데이터 분석 (15분)

시나리오: 조사 결과를 바탕으로 비교 분석표를 만든다

4개 조사 파일을 모두 읽고 비교 분석하라.
_workspace/research_eu.md
_workspace/research_us.md
_workspace/research_kr.md
_workspace/research_asia.md

다음을 수행하라:

1. 국가별 비교표를 만들어라
   | 항목 | EU | 미국 | 한국 | 일본 | 중국 |
   - 규제 방식 (법률/행정명령/자율)
   - 고위험 AI 정의
   - 벌금/제재 수준
   - 시행 시기
   - 언론/미디어 관련 조항

2. 한국 AI 기본법이 해외 법안 대비 
   강한 점과 약한 점을 각각 3가지씩 분석하라.

3. 기사 앵글 3개를 제안하라.
   각 앵글에 헤드라인(안)과 리드 1문장을 포함.

결과를 _workspace/analysis_ai_regulation.md에 저장.

[시연 포인트]

  • 에이전트가 4개 파일을 한 번에 읽고 교차 분석
  • 사람이 일일이 비교할 필요 없이 구조화된 비교표 생성
  • 기사 앵글까지 자동 제안

참가자 실습:

  • 7-1에서 생성된 자기 조사 파일들을 분석 에이전트에게 전달
  • 비교 분석표 + 기사 앵글 생성

7-3. 실습 L: 에이전트로 기사 초안 작성 (15분)

시나리오: 분석 결과를 토대로 기사 초안을 생성한다

다음 자료를 읽고 해설 기사 초안을 작성하라.

입력 파일:
- _workspace/analysis_ai_regulation.md (비교 분석)
- _workspace/research_kr.md (한국 상세)

기사 조건:
- 형식: 해설 기사, 2000자 내외
- 앵글: "한국 AI 기본법, 세계와 비교하면 어디쯤인가"
- 구조:
  [리드] 핵심 팩트 1문장 + 왜 중요한지 1문장
  [본문1] 글로벌 AI 규제 지형 요약 (비교표 활용)
  [본문2] 한국 AI 기본법의 특징 — 해외 대비 강점/약점
  [본문3] 언론/미디어에 미치는 영향
  [마무리] 남은 과제와 전망
- 톤: 설명적, 드라이, 경어체
- 모든 수치와 사실에 출처를 [괄호]로 표기
- 전문가 인용이 필요한 곳은 [인용 필요: 분야]로 표기
- "결국", "본질은", "핵심은" 사용 금지

결과를 _workspace/draft_ai_regulation.md에 저장.

[시연 포인트]

  • 에이전트가 조사→분석→초안의 전체 맥락을 이해하고 일관된 기사를 작성
  • [인용 필요], [확인 필요] 태그로 기자가 보완할 부분을 명확히 표시
  • 초안을 받은 뒤 기자가 편집자로서 판단하는 과정이 핵심

참가자 실습:

  • 자신의 분석 결과를 토대로 기사 초안 생성
  • AI 초안을 직접 읽으며 편집 포인트 3가지 이상 메모

7-4. 전체 파이프라인 회고 + 토론 (5분)

방금 경험한 것을 정리하면:

[사람] 이슈 선정 + 앵글 결정
  ↓
[에이전트 ×4] 병렬 자료 조사 (5~10분)
  ↓
[에이전트] 교차 분석 + 비교표 (3~5분)
  ↓
[에이전트] 기사 초안 (3~5분)
  ↓
[사람] 편집, 팩트체크, 취재원 인터뷰, 최종 판단

사람이 하는 일 vs AI가 하는 일:

사람 (대체 불가) AI 에이전트 (보조)
뉴스 가치 판단 대량 자료 수집
앵글 결정 교차 비교 분석
취재원 인터뷰 구조화된 초안
팩트체크 최종 확인 패턴 탐지
윤리적 판단 형식 변환
발행 결정 —

토론 질문:

  • "이 파이프라인에서 AI를 빼면 어떤 일이 얼마나 걸리는가?"
  • "AI가 틀릴 수 있는 지점은 어디인가?"
  • "이 워크플로우를 우리 뉴스룸에 적용한다면?"

부록: 프롬프트 템플릿 12선

참가자가 바로 복사해서 쓸 수 있는 프롬프트 모음.

1. 기사 요약

다음 기사를 읽고 아래 형식으로 요약하라.
[기사 붙여넣기]
- 핵심 사실 (1~2문장): 누가, 무엇을, 왜
- 배경 맥락 (2~3문장): 왜 지금 중요한가
- 이해관계자: 찬성 / 반대 입장 (각 1문장)
- 후속 전개: 앞으로 주목할 포인트
조건: 원문에 없는 정보를 추가하지 마라.

2. 인터뷰 질문 생성

나는 [주제]에 대해 [이름/직함]을 인터뷰한다.
배경: [최근 발언/행동], [관련 이슈 현황]
질문 10개를 만들어라:
- 도입 2개, 핵심 5개, 도전적 2개, 마무리 1개
각 질문에 의도를 병기하라.

3. 보도자료 → 기사

다음 보도자료를 800자 스트레이트 기사로 작성하라.
[보도자료]
조건: 역피라미드, 홍보성 표현 제거, 주어 명확히,
빠진 정보 [확인 필요], 반대 의견 [보완 필요] 표기.

4. 데이터 분석

첨부 CSV를 분석하라.
1. 개요 (행/열, 기간, 변수)
2. 기술 통계 (평균, 중위값, 최대/최소)
3. 추이 분석 (시간에 따른 변화)
4. 이상치 탐지
5. 기사 앵글 3개 제안
수치를 포함하라. 인과관계를 단정짓지 마라.

5. 팩트체크

다음 주장을 검증하라: "[주장]"
1. 핵심 팩트 분리
2. 각 팩트의 근거와 출처
3. 반박 근거
4. 판정: 사실/대체로 사실/절반의 사실/거짓

6. 녹취록 정리

다음 인터뷰 녹취록을 정리하라.
[녹취록]
1. 주제별로 발언을 분류하라
2. 핵심 발언 5개를 원문 그대로 추출하라
3. 각 발언의 맥락을 1문장으로 요약하라
발언을 수정하거나 의역하지 마라.

7. 해설 기사 구조화

다음 이슈에 대한 해설 기사 구조를 잡아달라: [이슈]
1. 무엇이 일어났나 (사실)
2. 왜 일어났나 (배경)
3. 왜 중요한가 (영향)
4. 앞으로 어떻게 될까 (전망)
5. 남은 질문 (추가 취재 포인트)
각 항목에 확인해야 할 팩트와 필요한 취재원을 제안하라.

8. 뉴스레터 변환

이 기사를 뉴스레터 도입부(200자)로 바꿔라.
독자에게 말을 거는 톤, "왜 당신에게 중요한지"로 시작.
3가지 버전을 만들어라.

9. 비교 분석

다음 두 문서/입장을 비교 분석하라.
[문서 A] [문서 B]
1. 공통점 (동의하는 부분)
2. 차이점 (대립하는 부분) — 표로 정리
3. 각 측의 논거 강도 평가
4. 빠져있는 관점

10. AI 산출물 자체 검증

방금 네가 작성한 내용을 스스로 검증하라.
1. 확실한 사실과 불확실한 추정을 구분하라
2. 출처가 필요한 주장을 표시하라
3. 과장되거나 단정적인 표현을 찾아 수정하라
4. 빠뜨린 중요한 관점이 있다면 추가하라

11. 공공데이터 탐색

[주제]에 대한 기사를 쓰려고 한다.
활용할 수 있는 한국 공공데이터를 추천하라.
각 데이터에 대해:
- 데이터 이름과 출처 (URL)
- 포함 항목
- 갱신 주기
- 기사에 활용하는 방법
공공데이터포털(data.go.kr), 통계청(kostat.go.kr) 등을 우선 확인하라.

12. 다각도 앵글 생성

다음 사안에 대해 서로 다른 앵글의 기사 5개를 제안하라: [사안]
각 앵글에 대해:
- 헤드라인 (안)
- 리드 1문장
- 핵심 취재 포인트
- 필요한 취재원
기존에 많이 다뤄진 뻔한 앵글은 제외하고, 독자적 시각을 제안하라.
